Operating Systems Terminology

TERM DEFINATION
Operating System An Operating System is the software that controls the overall operation of a computer. It provides the means by which a user can store and retrieve files, provides the interface by which a user can request the execution of programs, and provides the environment necessary to execute the programs requested.
Job Queue In batch processing systems, the jobs residing in mass storage wait for execution in a Job Queue. A Queue is a storage organization in which objects (in this case, jobs) are ordered in FIFO.
FIFO FIFO (First-in, First-out) is, the objects are removed from the queue in the order in which they arrived. In reality, most job queues do not rigorously follow the FIFO structure, since most operating systems provide for consideration of job priorities. As a result, a job waiting in the job queue can be bumped by a higher-priority job.

Time-Sharing Time-Sharing is an operating system or software environment that allows multiple users to interact with and use a computer system simultaneously. Each user is given the illusion that they have exclusive access to the machine, even though resources are being shared.
Multiprograming Multiprograming is a computer system management technique where multiple programs or processes are loaded into memory simultaneously and executed concurrently. In multiprogramming, the CPU's time is divided into small intervals or time slices, and during each time slice, a different program or job is allowed to execute.
Application Software Application Software , often simply referred to as "applications" or "apps," is a category of computer software that is designed and developed to perform specific tasks or functions for end-users. It is distinct from system software, which includes the operating system and utility programs that manage and maintain the computer's hardware and provide a platform for application software to run.
System Software System Software refers to a category of computer software that is responsible for managing and controlling the fundamental operations and resources of a computer system. It acts as an intermediary between the hardware and application software, facilitating the efficient functioning of the computer.
Utility Software Utility Software, within the context of system software, refers to a set of specialized programs and tools designed to perform specific tasks that help manage and optimize a computer system. Utility software is typically used to enhance system performance, improve security, and streamline various maintenance and administrative tasks.
Graphical User Interface (GUI) A Graphical User Interface (GUI) is a type of user interface that allows users to interact with a computer or software application through graphical elements such as icons, windows, menus, and buttons, as opposed to text-based or command-line interfaces. In a GUI, users can perform tasks and issue commands by pointing and clicking with a mouse or by tapping and swiping on a touchscreen device.
Shell A Shell is a software program that serves as an interface between a user and an operating system. It allows users to interact with the operating system by accepting textual commands or scripts entered through a keyboard and displaying text-based responses on a monitor or screen.
Kernel Kernel is the central and core component of an operating system. It is the fundamental part of the operating system responsible for managing and controlling the computer's hardware and providing essential services for software applications. The kernel interacts directly with the computer's hardware and serves as an intermediary between the hardware and software layers.
Device Driver Device Driver are essential for the proper functioning of hardware devices within a computer system. They bridge the gap between hardware and software, allowing the operating system and user applications to interact with and utilize hardware components effectively. Without device drivers, the computer would not be able to utilize its hardware components, making them essentially non-functional.
File Manager A File Manager, within the context of an operating system, is a software component responsible for organizing, managing, and coordinating the use of the computer's mass storage facilities, which includes hard drives, solid-state drives, external storage devices, and other storage media. Its primary role is to facilitate the efficient storage, retrieval, and manipulation of files and directories on these storage devices.
Memory Manager Memort Mamager is a crucial component of an operating system's kernel responsible for coordinating the efficient utilization of a computer's main memory (RAM). Its primary function is to manage the allocation and deallocation of memory resources, ensuring that multiple programs can run concurrently, share memory, and that each program can access its allocated memory space without conflicts.
Virtual Memory Virtual Memory is a computer memory management technique and a system feature that extends a computer's physical memory (RAM) by using a portion of the computer's storage, typically on a hard drive or solid-state drive, as an extension of RAM. Virtual memory allows a computer to execute processes or run applications that require more memory than the physical RAM can provide.
Boot Strapping Boot Strapping, in the context of computing and computer science, refers to the process of initializing a computer system or software program. It involves the loading of a small, essential program, typically the operating system, into a computer's memory from a non-volatile storage device (such as a hard drive or flash memory) in order to start the computer and make it operational.
ROM ROM(Read-only Memory) is a type of computer memory that is used to store non-volatile, permanent data and instructions that are essential for the operation of a computer or electronic device. ROM retains its data even when the power is turned off, making it a reliable means of storing critical information that is needed during the bootstrapping process and for the functioning of the device.
Deadlock Deadlock is a state in computer science and operating systems where two or more processes or threads are unable to proceed because each is waiting for a resource that is currently held by another, causing a circular dependency and preventing any further progress. In a deadlock situation, the processes involved are effectively stuck, and the system cannot make any forward progress.
Spooling Spooling is a computer technique used to manage and coordinate input and output (I/O) operations in a way that improves the efficiency of data processing. It involves temporarily holding data in a spool (buffer) before it is processed, enabling multiple processes or devices to access and work with a shared resource more effectively.
Super User(or Administrator) The term "Super User" or "Administrator" refers to a user account with the highest level of privileges and authority within a computer system or network. This user account is typically assigned during system setup and is granted extensive control and access rights, often including the ability to make system-wide changes, configure settings, install and uninstall software, and perform administrative tasks.
Multi-Factor Authentication Multi-Factor Authentication (MFA), sometimes referred to as Two-Factor Authentication (2FA), is a security method that requires users to provide multiple forms of evidence, or "factors," to confirm their identity before granting access to a computer system, online account, or a secure resource.
