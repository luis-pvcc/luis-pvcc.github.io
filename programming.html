Chapter 5: Programming Algorithms & Software Testing

Phases of computer program development:

  1. Understand the problem
  2. Design an algorithm to solve the problem
  3. Write the program code in an appropriate language based on the algorithm
  4. Enter the program and save
  5. Execute the program
  6. Test the program, find errors and debug, then retest
  7. Document and evaluate the program

Programming structures:

  • Sequential structure: Top-down execution
  • Decision structures: if/then/else
  • Iterative (Looping):
    • while loop: continue looping as long something is true
    • repeat/until loop: continue looping until something is true
    • for loop: loop a given number of times
  • Recursive structures:
    • Repeat a set of instructions as a subtask of itself
    • Repetition continues until a degenerative case (base case) is met

Two classic programming/database problems:

  • Sorting:
    • Descending order/Ascending order
    • Sorting numbers
    • Sorting strings (in accordance with ASCII values)
  • Searching: find specific data that meets a certain criteria
    • In database:
      • Filters: temporary search
      • Queries: search criteria are saved so they don't have to be recreated
    • Two popular search techniques:
      • Sequential search: search from top to bottom or bottom to top, one item at a time
      • Binary search: only works if data is SORTED first, start in the middle, then go to half of that, etc.

Two classic program goals:

  • Efficiency: Program takes minimal time and/or resources
  • Correctness: Program runs in expected ways and works for all expected conditions

Software testing and verification

  • Kinds of tests:Information Link
    • Testing Type Definition
    Acceptance testing Acceptance testing is a type of software testing that evaluates whether a system satisfies the acceptance criteria and meets the specified requirements of the end-users or stakeholders.
    Integration testing Integration testing is a software testing technique where individual components or modules of a system are combined and tested as a group to ensure they work together correctly.
    Unit testing Unit testing is the practice of testing individual units or components of a software application in isolation to ensure their correctness and reliability.
    Functional testing Functional testing is a type of software testing that evaluates the functionality of a system by testing it against the specified requirements and use cases.
    Performance testing Performance testing involves evaluating the speed, responsiveness, and scalability of a software system under various conditions to ensure it meets performance requirements.
    Regression testing Regression testing is the process of retesting a software application to verify that new code changes or updates do not introduce defects or negatively impact existing functionality.
    Stress testing Stress testing is a type of testing that assesses the robustness and stability of a system under extreme conditions or high loads to identify potential weaknesses or vulnerabilities.
    Usability testing Usability testing focuses on evaluating how user-friendly and intuitive a software application is by observing real users as they interact with the system.
    Architectural flaws Software testing is done to detect architectural flaws in the system's design, which may impact the system's stability, scalability, and maintainability.
    Poor design decisions Software testing helps identify poor design decisions that can lead to inefficiencies, maintenance challenges, and overall system problems.
    Invalid or incorrect functionality Testing is conducted to find invalid or incorrect functionality in the software that deviates from the specified requirements or user expectations.
    Security vulnerabilities Software testing is essential for uncovering security vulnerabilities and weaknesses that could expose the system to threats and attacks.
    Scalability issues Testing helps identify scalability issues in software that may affect its ability to handle increased loads or user demands effectively.
    Testing best practices Testing best practices encompass a set of recommended approaches and techniques for achieving high-quality software, including continuous testing, configuration management, service virtualization, defect tracking, and metrics and reporting.
    Term Definition
    Testing best practices Best practices in software testing refer to the recommended approaches, methodologies, and strategies for testing software applications. These practices help ensure the quality, reliability, and efficiency of the software development and testing processes.
    Software testing and verification section The software testing and verification section typically focuses on the activities related to testing and verifying software products. It includes test planning, test case design, test execution, and validation to ensure that software meets its intended requirements and functions correctly.
    Continuous testing Continuous testing is an approach where testing is integrated into the software development process and occurs continuously throughout the development lifecycle. It aims to detect and address issues early in the development process, ensuring that software is always in a testable state.
    Configuration management Configuration management involves tracking and controlling changes to software and related documents, ensuring that different versions and configurations are well managed. It helps maintain the integrity and consistency of software throughout its lifecycle.
    Service virtualization Service virtualization is a testing technique that involves simulating the behavior of dependent services, systems, or components that may not be readily available for testing. It allows testers to replicate the behavior of these services for more comprehensive testing.
    Defect or "bug tracking" Defect or "bug tracking" is the process of documenting, monitoring, and managing software defects or issues that are identified during testing. This helps prioritize and resolve defects to improve software quality.
    Metrics and reporting Metrics and reporting involve collecting and analyzing data related to software testing and quality. These metrics provide insights into the effectiveness of testing processes and help in making data-driven decisions for improvement.
